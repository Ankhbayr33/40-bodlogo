1. Бодлогын утга:
Бидэнд n ширхэг өндөр бүхий баганууд өгөгдсөн. Жишээ нь h = [2, 1, 5, 6, 2, 3] гэж үзвэл, эдгээрийг график хэлбэрээр байрлуулбал багана бүрийн дор тэгш өнцөгт гарч ирнэ. Бид эдгээр багануудыг залган хамгийн том тэгш өнцөгт талбайг тооцоолох хэрэгтэй.

2. Хэрхэн шийдсэн:
Бид уламжлалт brute-force буюу хоёр давхар давталт ашиглавал хугацааны хувьд O(n^2) болох ба энэ нь 10⁵ зэрэг хэмжээтэй оролтод тохирохгүй. Тиймээс Stack өгөгдлийн бүтэц ашиглан O(n) хугацаанд шийдэх боломжтой алгоритмыг ашигласан.

3. Stack ашигласан гол санаа:
Stack нь баганын индексүүдийг хадгална.

Stack дотор байгаа индексүүдийн өндөр нь өсөх дарааллаар байна.

Өсөлттэй байгаа баганууд үргэлжилж байх үед бид шууд Stack руу хийдэг.

Гэтэл шинэ багана өмнөхөөс намхан байвал бид талбайг тооцоолж эхэлнэ.

4. Талбайг хэрхэн тооцдог вэ?
Stack-ээс гаргасан багана дээр тулгуурлаж:

Тухайн багана i-ээс өмнөх хамгийн өндөр тэгш өнцөгтийг тооцно.

Өндөр = гаргаж байгаа баганын өндөр

Өргөн = i - previous_index - 1, previous_index нь stack доторх дараагийн элемент

Энэ тооцооллоор тухайн өндөр дээр суурилсан хамгийн том талбайг олно.

Ийм байдлаар үргэлжлүүлэн явна.

5. Эцэст үлдсэн stack-ийн элементүүдийг шалгах:
Stack-д байгаа үлдсэн индексүүд нь бас өндөр өсөлттэй дараалалтай тул дээрхийн адил процессоор бүрэн тооцоолж дуусгана.

Бодлогын жишээ:
Жишээ оролт: [2, 1, 5, 6, 2, 3]

5, 6 дээр тулгуурлан өндөр 5, өргөн 2 → талбай = 10

2 дээр тулгуурлан өндөр 2, өргөн 4 → талбай = 8

Хамгийн их талбай = 10

Алгоритмын хугацаа ба санах ой:
Хугацааны нийлбэр = O(n) — Учир нь багана бүрийг зөвхөн нэг удаа stack руу хийж, нэг удаа гаргана.

Санах ойн зарцуулалт = O(n) — Stack-д хадгалагдаж буй индексүүдийн тоо n-ээс хэтрэхгүй.

Дүгнэлт:
Энэ бодлогыг Stack ашиглан хамгийн бага хугацаанд (O(n)) гүйцэтгэсэн. Stack нь өсч буй өндөртэй багануудыг цувуулан хадгалж, намхан багана гарах үед өмнөх өндөртэй баганууд дээр суурилж талбайг тооцоолж байгаагаараа үр ашигтай шийдэл болдог. Бодлогын гол санаа нь тухайн баганаас аль тал руу хэдэн ширхэг багана үргэлжилж байгааг хурдан тооцоолж чадах явдал юм.
